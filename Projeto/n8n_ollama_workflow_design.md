# 🤖 Workflow n8n para SIAP Automator com Ollama

Este documento detalha a arquitetura de um workflow no n8n para integrar o SIAP Automator com um modelo de linguagem grande (LLM) rodando localmente via Ollama. O objetivo é automatizar a geração de conteúdo educacional, como planejamentos de aula e diários, aproveitando a privacidade e o controle de um ambiente local.

## 1. Visão Geral da Arquitetura

O workflow será acionado pela extensão SIAP Automator, que enviará requisições para um webhook no n8n. O n8n, por sua vez, utilizará um nó do Ollama para interagir com um LLM local, gerando o conteúdo solicitado e retornando-o para a extensão.

```mermaid
graph TD
    A[Extensão SIAP Automator] --> B{Webhook n8n}
    B --> C{Switch: Tipo de Tarefa}
    C -- Planejamento --> D[Ollama Node: Gerar Planejamento]
    C -- Diário --> E[Ollama Node: Gerar Diário]
    D --> F[Set Node: Formatar Saída]
    E --> F
    F --> G[Webhook Response]
    G --> A
```

**Componentes Principais:**

*   **Extensão SIAP Automator**: Envia dados da página do SIAP para o n8n e recebe o conteúdo gerado.
*   **Webhook n8n**: Ponto de entrada para as requisições da extensão.
*   **Nó Switch**: Direciona a requisição para o fluxo de automação correto (planejamento ou diário) com base no `taskType`.
*   **Nó Ollama**: Interage com o modelo de linguagem local (ex: Llama 3.1, Mistral-Nemo) para gerar o conteúdo.
*   **Nó Set**: Formata a resposta do LLM para que a extensão possa processá-la facilmente.
*   **Webhook Response**: Envia a resposta final de volta para a extensão.

## 3. Prompts de Sistema para o Ollama

Os prompts de sistema são cruciais para guiar o comportamento do LLM e garantir que ele gere o conteúdo educacional de forma adequada. Eles serão configurados no nó Ollama do n8n.

### 3.1. Prompt para Geração de Planejamento

Este prompt será usado quando a extensão solicitar a geração de um planejamento de aula. Ele deve instruir o LLM a criar um planejamento detalhado com habilidades, objetivos, metodologias, avaliação e recursos, com base nos dados da aula e em orientações extras fornecidas.

```
Você é um assistente de planejamento educacional especializado em criar planos de aula detalhados para professores. Sua tarefa é gerar um planejamento de aula completo, incluindo habilidades, objetivos de conteúdo, metodologias, avaliação e recursos didáticos, com base nas informações fornecidas.

**Informações da Aula:**
- Ano Letivo: {{ $json.body.aula.anoLetivo }}
- Composição de Ensino: {{ $json.body.aula.composicaoEnsino }}
- Série: {{ $json.body.aula.serie }}
- Turno: {{ $json.body.aula.turno }}
- Componente Curricular: {{ $json.body.aula.componenteCurricular }}
- Turma: {{ $json.body.aula.turma }}
- Número da Aula: {{ $json.body.aula.numeroAula }}

**Orientações Extras:**
{{ $json.body.conteudoExtra || 'Nenhuma orientação extra fornecida.' }}

**Formato da Saída:**
Forneça o planejamento em um formato estruturado, separando claramente cada seção. Use os seguintes cabeçalhos:

**Habilidades:**
[Liste as habilidades aqui]

**Objetivos de Conteúdo:**
[Liste os objetivos aqui]

**Metodologias:**
[Descreva as metodologias aqui]

**Avaliação:**
[Descreva a avaliação aqui]

**Recursos Didáticos:**
[Liste os recursos aqui]

Certifique-se de que o conteúdo seja relevante para o componente curricular e a série, e que incorpore as orientações extras quando aplicável. Seja conciso, mas completo.
```

### 3.2. Prompt para Geração de Conteúdo do Diário

Este prompt será utilizado para gerar o conteúdo do diário de aula, que geralmente é um resumo do que foi ministrado, atividades realizadas e observações relevantes. Ele pode ser mais direto, focando na síntese do conteúdo.

```
Você é um assistente educacional que ajuda professores a registrar o conteúdo ministrado em suas aulas. Sua tarefa é gerar um resumo conciso e informativo para o diário de aula, com base nas informações fornecidas.

**Informações da Aula:**
- Ano Letivo: {{ $json.body.aula.anoLetivo }}
- Componente Curricular: {{ $json.body.aula.componenteCurricular }}
- Turma: {{ $json.body.aula.turma }}
- Número da Aula: {{ $json.body.aula.numeroAula }}

**Tópicos Abordados / Atividades Realizadas:**
{{ $json.body.topicos || 'Nenhum tópico específico fornecido. Gere um resumo geral com base na aula.' }}

**Formato da Saída:**
Forneça um parágrafo ou uma lista de pontos que descreva o conteúdo principal da aula, as atividades desenvolvidas e, se houver, observações importantes. O texto deve ser adequado para um registro de diário de classe.
```

## 4. Configuração do Nó Ollama no n8n

Ao configurar o nó Ollama no n8n, será necessário especificar o modelo a ser utilizado (ex: `llama3.1`, `mistral-nemo`) e o host onde o Ollama está rodando (geralmente `http://localhost:11434`). A temperatura do modelo pode ser ajustada para controlar a criatividade da resposta; um valor mais baixo (ex: 0.1-0.5) é recomendado para maior consistência em conteúdos educacionais.
